## 二、电源方案（核心改动）

> **问题背景**：开关附近没有电源插座，需要电池供电。

### 2.1 系统功耗分析

| 模块 | 工作状态 | 电流 |
|------|----------|------|
| STM32F103 | 运行 | 20~50mA |
| JDY-16 BLE 蓝牙 | 空闲/连接 | 5~15mA |
| MG90S 舵机 | 静止（耗电极小） | <5mA |
| MG90S 舵机 | **动作瞬间** | **200~500mA**（峰值） |
| **总计** | **空闲时** | **约 55~135mA** |
| | **舵机动作瞬间** | **约 250~580mA** |

### 2.2 推荐供电方案：18650 锂电池升压充电一体模块

> 🚀 **新方案**：使用 IP5306/IP2312 方案的 **18650 升压充电一体板**，替代原方案中的 TP4056 + MT3608 + 电池盒 三个独立模块。一块板子搞定充放电管理、升压到 5V 和 3.3V、过充过放过流保护。

```
电源链路（简化后）：
[USB 充电头] → [一体板 Micro-USB] → [IP5306 管理芯片] → [18650 电池]
                                                          ↓
                                                  升压至 5V → STM32 (5V 引脚) + 舵机 VCC
                                                  降压至 3.3V → JDY-16 VCC
```

**为什么选这个方案：**
- **接线极简**：电源部分只需接 4 根线（电池正负极、5V 输出、3.3V 输出），比原方案少 6 根线
- **免调试**：出厂固定 5V 输出，无需万用表调电压
- **功能齐全**：一块板集成充电管理 + 电池保护（过充/过放/过流/短路）+ 升压到 5V + 降压到 3.3V
- **空间更小**：一块 PCB 约 4×2×1cm，比分体式三个模块小 2/3，容易藏进外壳
- **成本更低**：一体板 ¥12~18，比分体式（TP4056 ¥3 + MT3608 ¥5 + 电池盒 ¥3）**更便宜**
- **更安全**：少一个接线环节就少一个故障点，不存在「P+ 和 B+ 接反」「两路 5V 冲突」等风险

> ⚠️ **关于供电切换（调试/烧录时）**：
> - **正常使用 + 烧录调试**：一体板 5V 输出始终供电给 STM32 的 5V 引脚，**烧录时不需要断开电池**。
> - **ST-Link 通过 SWD 接口（SWCLK/SWDIO/3.3V/GND）连接 STM32，不走 USB 口供电**，所以不存在两路 5V 冲突的问题。
> - **无需每次烧录都拔电池或拔杜邦线**，接好 ST-Link → Keil 按 F7 编译 → F8 烧录 → 完成，全程不断电。

### 2.3 一体板使用步骤

**接线极其简单，只需 4 根线（见 3.1 节接线表）**，步骤如下：

1. **焊接电池连接线**（仅首次需要）
   - 大多数一体板出厂没有焊电池线，需要自己焊
   - 把 18650 电池正极（红色线）焊到 BAT+ 焊盘，负极（黑色线）焊到 BAT- 焊盘
   - 如果没有电烙铁，可以买**带引出线的 18650 电池盒**代替焊接

2. **接充电口（Micro-USB/Type-C）**
   - 用普通手机数据线连接一体板 Micro-USB 口和 USB 充电头（5V/1A 即可）
   - 充电时板上 LED 亮起，充满后 LED 熄灭或变绿

3. **接负载**
   - 一体板的 **5V+/5V-** 输出 → 舵机红线 + STM32 5V 引脚
   - 一体板的 **3.3V+/3.3V-** 输出 → JDY-16 VCC
   - **所有 GND 共地**（5V- 和 3.3V- 内部已连通）

   **杜邦线连接细节：**
   - 大多数一体板的输出引脚是 **2.54mm 间距排针**（已焊好）或 **焊盘**（需要自己焊排针）
   - 如果是排针：用 **母对母杜邦线**（一头套排针，另一头接 STM32/舵机排针）
   - 如果是焊盘：先焊一排排针上去（¥1 一排 40Pin），再用杜邦线连
   - STM32 开发板和舵机的引脚都是排针，直接用杜邦线插就行
   - **建议买一盒公对公 + 公对母 + 母对母三种杜邦线混装**（¥5~8），整个项目都够用

4. **加电容建议**（可选，推荐）
   在 5V 输出端加一个 **100µF/25V 电解电容**，防止舵机启动瞬间大电流拉低电压导致 JDY-16 重启。

   ```
   一体板 5V+ ────┬──── 给负载
                   │
                  [100µF / 25V 电解]
                   │
   一体板 GND ─────┴──── 所有 GND
   ```

> ✅ **一体板不需要万用表调电压**，出厂就是固定 5V 输出。
> ✅ **不需要额外保护板**，一体板自带过充/过放/过流/短路保护。

5. **充电时能同时使用吗？（常见疑问）**
   - **可以，完全没问题。** IP5306/IP2312 方案的一体板支持**边充边放**（pass-through charging）
   - 插着 USB 充电的同时，舵机、STM32、JDY-16 都能正常工作
   - 充电时电池在充电，负载由充电电路直接供电，不会冲突
   - **唯一注意**：如果充电头功率太小（< 5V/1A），充电 + 舵机动作时可能出现电压不稳，建议用 **5V/2A 充电头**

### 2.4 续航估算

| 电池容量 | 空闲电流（改进前） | 空闲电流（改进后） | 估算续航 | 适用场景 |
|----------|----------|----------|----------|----------|
| 2000mAh | ~80mA | ~50mA（禁用空闲外设+降低主频） | **约 40 小时** | 每天开关 20 次左右 |
| 3500mAh | ~80mA | ~50mA | **约 70 小时** | 每天开关 10 次左右，更久 |

> 舵机动作瞬间电流大但时间很短（约 0.3 秒），对续航影响很小。如果电池不够用，可以买两节 18650 换 **双节串联**（7.4V）+ 改用 **LM2596 降压模块** 降到 5V，但成本和复杂度会更高，先不考虑。
> 
> 💡 **降低空闲功耗的技巧**（在 CubeMX 中配置）：
> - 在 CubeMX 的 **Clock Configuration** 页，将 HCLK 从默认 72MHz 降到 **48MHz**（够用，省电约 30%）
> - 在 CubeMX 中**不要启用不需要的外设**（CubeMX 会自动门控未配置外设的时钟，无需手动禁用）
> - 代码中加入 `HAL_SuspendTick()` 在空闲时停止 SysTick

| 一体板端子 | 接什么 |
|------------|--------|
| **5V+** | 舵机红线 + STM32 的 **5V 引脚** |

## 四、阶段 2：CubeMX 配置（第 1~2 天，预计 2~3 小时）

### 4.1 新建工程

1. 打开 **STM32CubeMX**
2. 点击 **New** → 新建工程
3. 在搜索框输入 `STM32F103C8` → 选择 `STM32F103C8Tx` → 点 **Start Project**

### 4.2 配置 RCC（外部晶振）

1. 左侧菜单：**System Core → RCC**
2. 点击 **HSE**，下拉选择 **Crystal/Ceramic Resonator**（晶振）
   - 不选这个会导致定时器时钟不准，舵机角度错误

### 4.3 配置调试接口

1. 左侧菜单：**System Core → SYS**
2. **Debug/Serial** 下拉选择 **Serial Wire**
   - 允许通过 ST-Link 调试和烧录

### 4.4 配置 TIM2（PWM 输出控制舵机）

1. 在右侧芯片引脚图上找到 **PA0** → 点击它
2. 弹出菜单中选择 **TIM2_CH1**
3. 左侧出现 TIM2，进入 **Timers → TIM2 → Parameter Settings**：
   - **Counter Mode**：Up
   - **Counter Period (Auto-Preload)**：`19999`（PWM 周期 = 定时器时钟 / (19999+1) = 1MHz / 20000 = 50Hz）
   - **Prescaler**：`71`（72MHz 经分频 → 72MHz / (71+1) = 1MHz 定时器时钟）
   - **Channel 1 PWM Configuration**：
     - **PWM Mode**：PWM Mode 1
     - **Pulse**：`1000`（初始值，后面在代码里改）
     - **Output State**：Enable
     - **Polarity**：High
4. **Clock Configuration**（时钟树）：
   - 点击顶部 **Clock Configuration** 标签
   - 找到 **HCLK**，输入 `72`，按回车确认
   - 确认所有连线都是灰色（灰色 = 配置成功）

### 4.5 配置 USART1（蓝牙串口通信）

1. 在引脚图上找到 **PA9** → 点击 → 选择 **USART1_TX**
2. 找到 **PA10** → 点击 → 选择 **USART1_RX**
3. 左侧进入 **Connectivity → USART1 → Parameter Settings**：
   - **Mode**：Asynchronous
   - **Baud Rate**：`115200`
   - **Word Length**：8 Bits
   - **Parity**：No
   - **Stop Bits**：1
   - **Hardware Flow Control**：None

### 4.6 生成工程

1. **Help → Manage Embedded Software Packages** → 确保已安装 STM32F1 的固件包（如果没有，点 Install 安装）
2. 点击顶部 **Project Manager** 标签页：
   - **Project** 栏：填写 Project Name（如 `STM32_Switch`），Project Location 选全英文路径（如 `D:\STM32_Switch\`）
   - **Toolchain / IDE**：选择 **MDK-ARM V5**
3. 点击右上角 **GENERATE CODE** 按钮
4. 生成完成后弹窗点 **Open Project**，Keil 会自动打开工程

### 5.2 核心原理

舵机通过 PWM 信号的脉宽控制角度：

| 舵机位置 | 脉宽 | TIM2_CCR1 值（理论参考） | 对应操作 |
|-----------|------|---------------------------|----------|
| **UP（上推）** | 0.7ms ~ 1.0ms | 700 ~ 1000 | 关灯（杠杆上端推开关上沿） |
| **MID（中间）** | ~1.5ms | ~1500 | 杠杆水平，不接触开关 |
| **DOWN（下按）** | 2.0ms ~ 2.3ms | 2000 ~ 2300 | 开灯（杠杆下端按开关下沿） |

> **计算方法**：定时器时钟 = 72MHz / (PSC+1) = 72MHz / 72 = 1MHz，即每个 CCR 计数 = 1µs。因此脉宽(µs) = CCR 值。例如 1.5ms = 1500µs → CCR = 1500。

> ⚠️ 实际值需要在调试中确定（参考 7.2.3 Step 4），因为你的安装位置和杠杆长度会影响所需的舵机角度。**不要直接用 0° 和 180° 的极限值**，建议在 30°~150° 范围内调试。

> 这些值是理论参考值。实际可能需要微调 ±50~100，后面会说怎么调。

#### 5.3.1 在 `/* USER CODE BEGIN 1 */` 和 `/* USER CODE END 1 */` 之间添加：

```c
/* USER CODE BEGIN 1 */

#include <string.h>  // strcmp() 需要，部分 Keil 配置下不包含会报错

// 舵机位置对应的 CCR 值（理论参考，实际微调用）
// 定时器时钟 1MHz，每个 CCR = 1µs，舵机脉宽范围 500~2500µs
#define DUTY_UP    700  // 舵机向上推（关灯），脉宽 0.7ms（留出安全余量，避免 0° 极限堵转）
#define DUTY_MID   1500 // 杠杆水平不接触开关，脉宽 1.5ms（中间位置）
#define DUTY_DOWN  2300 // 舵机向下按（开灯），脉宽 2.3ms（留出安全余量，避免 180° 极限堵转）

// ===== 非阻塞舵机控制 =====
// 用状态机替代 HAL_Delay(300)，避免阻塞期间丢失串口数据
static uint32_t servo_move_start = 0;  // 舵机开始转动的时间戳
static uint8_t  servo_moving = 0;      // 舵机是否正在转动
static uint8_t  servo_return_mid = 0;  // 按压后是否需要自动回中
#define SERVO_MOVE_TIME    300         // 舵机转动等待时间(ms)
#define SERVO_RETURN_TIME  250         // 回中等待时间(ms)

// 设置舵机角度（非阻塞：立即返回，后台等待）
void Servo_SetAngle(int duty) {
    __HAL_TIM_SET_COMPARE(&htim2, TIM_CHANNEL_1, duty);
    servo_move_start = HAL_GetTick();
    servo_moving = 1;
    // 如果目标是 UP 或 DOWN（按压位置），标记需要回中
    servo_return_mid = (duty == DUTY_UP || duty == DUTY_DOWN) ? 1 : 0;
}

// 在 while(1) 中调用，检查舵机是否转到位，到位后自动回中
void Servo_Update(void) {
    if (!servo_moving) return;
    if (HAL_GetTick() - servo_move_start < SERVO_MOVE_TIME) return;

    // 舵机到位
    servo_moving = 0;

    if (servo_return_mid) {
        // 按压完成后自动回到中间位置，防止长时间堵转
        servo_return_mid = 0;
        __HAL_TIM_SET_COMPARE(&htim2, TIM_CHANNEL_1, DUTY_MID);
        servo_move_start = HAL_GetTick();
        servo_moving = 1;  // 继续等待回中完成
    }
}

// 蓝牙指令处理（支持单字符和多字符命令）
void Process_Command(uint8_t cmd) {
    static uint8_t cmd_buf[16];
    static uint8_t idx = 0;

    // 换行符/回车 → 处理完整指令
    if (cmd == '\n' || cmd == '\r') {
        // 跳过连续的 \r\n（串口工具通常发送 \r\n 两个字符）
        if (idx == 0) return;
        cmd_buf[idx] = '\0';
        idx = 0;

        if (strcmp((char*)cmd_buf, "1") == 0 || strcmp((char*)cmd_buf, "ON") == 0) {
            // 开灯：先转到 DOWN 位置，等舵机到位后自动回中
            Servo_SetAngle(DUTY_DOWN);
            HAL_UART_Transmit(&huart1, (uint8_t*)"CMD: ON (DOWN)\r\n", sizeof("CMD: ON (DOWN)\r\n") - 1, 100);
        }
        else if (strcmp((char*)cmd_buf, "2") == 0 || strcmp((char*)cmd_buf, "OFF") == 0) {
            // 关灯：先转到 UP 位置，等舵机到位后自动回中
            Servo_SetAngle(DUTY_UP);
            HAL_UART_Transmit(&huart1, (uint8_t*)"CMD: OFF (UP)\r\n", sizeof("CMD: OFF (UP)\r\n") - 1, 100);
        }
        // 调试命令（机械结构安装时使用）
        else if (strcmp((char*)cmd_buf, "TESTUP") == 0) {
            Servo_SetAngle(DUTY_UP);
            HAL_UART_Transmit(&huart1, (uint8_t*)"TEST: UP\r\n", sizeof("TEST: UP\r\n") - 1, 100);
        }
        else if (strcmp((char*)cmd_buf, "TESTDN") == 0) {
            Servo_SetAngle(DUTY_DOWN);
            HAL_UART_Transmit(&huart1, (uint8_t*)"TEST: DOWN\r\n", sizeof("TEST: DOWN\r\n") - 1, 100);
        }
        else if (strcmp((char*)cmd_buf, "SLEEP") == 0) {
            Servo_SetAngle(DUTY_MID);
            HAL_UART_Transmit(&huart1, (uint8_t*)"TEST: SLEEP (MID)\r\n", sizeof("TEST: SLEEP (MID)\r\n") - 1, 100);
        }
        return;
    }

    // 防溢出
    if (idx < sizeof(cmd_buf) - 1) {
        cmd_buf[idx++] = cmd;
    }
}

/* USER CODE END 1 */
```

#### 5.3.3 在 `while(1)` 循环中（`/* USER CODE BEGIN WHILE */` 和 `/* USER CODE END WHILE */` 之间）：

> ⚠️ **非阻塞接收 + ORE 保护**：下面的代码直接检查 RXNE 标志位轮询接收，并在每次读取前清除 ORE（溢出错误）标志，防止因字节未及时读取导致 USART 锁死。
>
> 更完善的做法（可选升级）是用 **UART 接收中断**（`HAL_UART_Receive_IT`） + 环形缓冲区，彻底消除漏字节风险。

```c
    while (1)
    {
        // 更新舵机状态机（非阻塞等待）
        Servo_Update();

        uint8_t rx_byte;
        // 清除可能的溢出错误（ORE），防止 USART 锁死
        if (__HAL_UART_GET_FLAG(&huart1, UART_FLAG_ORE) != RESET) {
            __HAL_UART_CLEAR_OREFLAG(&huart1);
        }
        // 非阻塞方式：直接检查 RXNE 标志位（有数据时为 SET）
        if (__HAL_UART_GET_FLAG(&huart1, UART_FLAG_RXNE) != RESET) {
            rx_byte = (uint8_t)(huart1.Instance->DR & 0xFF);  // 读取数据并清除标志
            Process_Command(rx_byte);
        }
    }
/* USER CODE END WHILE */
```

### 5.5 第一次测试（先用 USB 供电调试）

> ⚠️ **USART1 冲突警告**：STM32 蓝色小板上的 CH340（USB 转串口）和 JDY-16 蓝牙模块**都连接到 USART1（PA9/PA10）**。如果两个同时连接：
> - STM32 发送时（PA9）：信号同时发给 CH340 和 JDY-16 的 RX——这没问题（一发两收）
> - 接收时（PA10）：CH340 的 TX 和 JDY-16 的 TX **同时驱动同一条线**——会冲突，导致数据乱码或收不到
>
> **所以测试时必须二选一：**
> - **阶段 A（USB 串口调试）**：**先不接 JDY-16**（或只接 VCC/GND 不接 TX/RX），用电脑串口工具测试
> - **阶段 B（蓝牙测试）**：JDY-16 完整连接 PA9/PA10，通过手机蓝牙 APP 测试（此时电脑 USB 串口工具只能看到 STM32 发出的数据，不能可靠接收）
>
> **如果你买的蓝色小板 PA9/PA10 旁边有两组焊盘跳线（标有 TXD/RXD），可以挑断 CH340 那一侧的焊盘来永久断开 CH340 和 PA9/PA10 的连接。这样 JDY-16 就能独占 USART1，不会冲突。但断开后你将无法通过 USB 串口调试（只能用 ST-Link 或蓝牙调试）。新手建议先不挑断，按上面的二选一方法操作。**

1. 先用 USB 线给 STM32 供电，**暂时不接电池**
2. **先不接 JDY-16 模块**（或只接 JDY-16 的 VCC 和 GND，不接 TX/RX 信号线），避免与 CH340 冲突
3. 打开 **设备管理器**（右键"此电脑"→"管理"→"设备管理器"→"端口（COM 和 LPT）"），记下 STM32 对应的 COM 口号（如 `COM3`、`COM5`）
   > 如果设备管理器中没有出现新 COM 口，说明 **CH340 驱动没装**或 USB 线是充电线（无数据功能），请先安装 1.1 节的 CH340 驱动。
4. 打开 **XCOM**（串口工具），选择上一步查到的 COM 口，波特率 **115200**
5. 打开串口 → 应收到 `STM32 Ready`
6. **重要**：在 XCOM 的发送区域**勾选「发送新行」**（或在发送框中手动输入 `\r\n`），否则命令不会被处理（代码需要收到换行符才执行命令）
7. 发送 `1`（或 `ON`） → 舵机转到开灯位置（DUTY_DOWN）
8. 发送 `2`（或 `OFF`） → 舵机转到关灯位置（DUTY_UP）

## 六、阶段 4：蓝牙联调（第 3~4 天，预计 30 分钟）

### 6.1 连接 JDY-16（BLE 即开即用）

## 八、阶段 6：机械结构与安装（第 2 周后半，预计 2~3 天）

> 💡 **你的场景：** 出租屋开关是 **纵向翘板式**（一边按下，另一边翘起），设备从 **侧面** 安装。核心设计：舵机轴直接连接一根长杠杆，杠杆横跨整个开关表面，像跷跷板一样——舵机转 → 杠杆倾斜 → 一边压下一边抬起，完美匹配翘板开关的机械特性。

### 8.1 方案：跷跷板杠杆式（唯一方案）

| 原理 | 难度 | 可靠性 |
|------|------|--------|
| 舵机轴直接连接长杠杆，杠杆横跨开关表面；舵机旋转 → 杠杆倾斜 → 杠杆上端按压开关上沿（关灯）或杠杆下端按压开关下沿（开灯） | ⭐⭐ | ⭐⭐⭐⭐⭐ |

**为什么这个方案好：**
- 舵机的旋转运动直接转化为翘板开关的"一边下压、一边抬起"动作，**机械上天然匹配**
- 无需连杆、导轨、拨杆等中间传动件，**零磨损点**
- 杠杆越长，按压角度越平缓，对舵机堵转力矩要求越低
- 调试只需找两个角度值（上位和下位），极其简单
- 从侧面安装，不遮挡开关面板，不影响手动操作

---

### 8.2 跷跷板杠杆式 · 详细设计

#### 8.2.1 机械原理

```
        墙壁
   ┌──────────────┐
   │   [开关面板]  │
   │      ┌──┐    │    ← 上端（按下=关）
   │ 关→  │  │    │
   │      │  │    │    ← 翘板开关（纵向）
   │ 开→  │  │    │
   │      └──┘    │    ← 下端（按下=开）
   └──────┬───────┘
          │
     ┌────┴────┐
     │  长杠杆   │    ← 一根刚性长杠杆（约7~8cm），
     │ (跷跷板)  │       横跨整个开关正面。
     │  ┌──┐    │      中点固定在舵机输出轴上。
     │  │舵│    │      上端≈开关上沿，下端≈开关下沿。
     │  │机│    │
     │  └──┘    │      舵机转 → 杠杆倾斜 →
     └────┬─────┘      一端压入，一端抬起。
          │
      舵机本体  舵机固定座
```

**关键思路：** 舵机 **侧面安装**，旋转轴 **垂直于墙壁**（指向开关面板）。舵机输出轴上直接装一根 **长杠杆**（约 7~8cm），杠杆横跨开关的整个高度。杠杆的两端各粘一个橡胶垫片，接触开关表面。杠杆的中点固定在舵机输出轴上。

**运动逻辑：**
- 舵机顺时针转一个小角度 → 杠杆上端压向开关上沿 → 下沿自然翘起 → **关灯**
- 舵机逆时针转一个小角度 → 杠杆下端压向开关下沿 → 上沿自然翘起 → **开灯**

> ⚠️ 注意：舵机只转一个小角度（约 20°~40°）就够了，不需要转满 180°。实际角度取决于杠杆长度和开关行程。

#### 8.2.2 所需材料

| 物品 | 说明 | 获取方式 |
|------|------|----------|
| MG90S 舵机 | 金属齿轮，寿命长 | 已购 |
| **长杠杆** | 约 7~8cm 长，刚性，轻量化 | 建议：3D 打印 PLA / 碳纤维板 / 亚克力板 / 冰棍棒叠层胶合 |
| 舵机配套螺丝 | 用于将杠杆固定在舵机输出轴上 | 舵机包装内自带 |
| 橡胶/硅胶垫片 ×2 | 杠杆两端各粘一小块，防滑、不伤开关 | 旧鼠标垫剪一小块 |
| 3M 双面胶 / 热熔胶 | 固定舵机 + 外壳 | 五金店 |

**杠杆制作建议（按推荐度排序）：**

| 方案 | 材料 | 难度 | 说明 |
|------|------|------|------|
| 3D 打印 | PLA / PETG | ⭐⭐ | 最推荐。设计一个长条形件，中间开孔套舵机输出轴，两端加橡胶垫片 |
| 亚克力板切割 | 3mm 亚克力 | ⭐⭐⭐ | 需要线锯或激光切割，边缘打磨后很光滑 |
| 冰棍棒叠层 | 木棒 + 胶水 | ⭐ | 两层冰棍棒用胶水粘合，足够硬。中间钻小孔，用舵机配套螺丝固定 |
| 金属片加工 | 1mm 铝板/铁片 | ⭐⭐⭐ | 强度最高，但需要钻头打孔，边缘要打磨防割手 |

> **新手推荐**：先用冰棍棒方案做原型验证，角度调好后再 3D 打印正式版。

#### 8.2.3 组装步骤

**Step 1 — 制作杠杆**

取冰棍棒（或 3D 打印件 / 亚克力条），裁成约 7~8cm 长的条状。在正中间钻/烫一个小孔（直径约 2mm，刚好能套进舵机输出轴的齿轮齿纹）。如果担心强度，用两层冰棍棒叠粘。

在杠杆两端底部分别粘一小块橡胶垫片（约 5mm 直径，用旧鼠标垫剪）。

**Step 2 — 安装杠杆到舵机**

将舵机配套的十字/U 型摇臂取下不用。直接把自制的长杠杆套在舵机输出轴的齿轮上，用舵机包装里的金属螺丝锁紧固定。

```
        ┌──────────────┐
        │  开关面板     │
        │  ┌────────┐  │
        │  │  ┌──┐  │  │  ← 长杠杆横跨开关表面
        │  │  │  │  │  │
        │  │  │  │  │  │     ■ = 舵机输出轴位置（杠杆中点）
        │  │  └──┘  │  │
        │  └────────┘  │
        └──────┬───────┘
               │
            ┌──┴──┐
            │舵机  │
            └─────┘
```

**Step 3 — 确定安装位置**

把舵机贴在开关右侧（或左侧）墙壁上，使杠杆正好横跨在开关表面正前方约 1~2mm 处。舵机输出轴与开关中心对齐。

**关键尺寸：**
- 杠杆长度 ≈ 开关总高度 + 2×（橡胶垫片厚度）+ 少许余量
- 标准翘板开关高度约 7~8cm → 杠杆长度约 8~9cm（比开关略长，确保两端能压到）
- 舵机输出轴中心 ≈ 开关面板中心高度

**Step 4 — 临时固定调试**

先用双面胶把舵机粘在纸板上，纸板贴在开关旁边。接好电（USB 供电或电池）。通过手机发 `1`（开灯）/ `2`（关灯）让舵机转动，观察杠杆两端是否准确按压开关的两个位置。

**调试要点：**
- 如果杠杆上端按不准 → 上下微调舵机安装高度
- 如果按压力度不够 → 检查杠杆与开关表面的距离（太远按不动，太近会卡住）
- 如果两侧都按到但行程不对 → 调整舵机的 PWM 角度（`DUTY_UP` 和 `DUTY_DOWN`）
- **目标**：舵机转约 20°~40°，杠杆一端下沉 2~3mm 就能触发开关动作

**Step 5 — 确定最终参数**

调试找到合适角度后，记下两个 PWM 值（见第 5.3 节的 `DUTY_xxx`）：
```
DUTY_UP   = ××    ← 舵机顺时针转，杠杆上端压下（关）
DUTY_DOWN = ××    ← 舵机逆时针转，杠杆下端压下（开）
DUTY_MID  = 1500  ← 中间位置，杠杆水平，两端都不接触
```

**Step 6 — 最终固定**

调试满意后，用热熔胶或 3D 打印支架把舵机永久固定在开关侧面墙壁上。

### 8.5 常见问题 & 避坑

| 问题 | 原因 | 解决 |
|------|------|------|
| 杠杆压不动开关 | 杠杆离开关表面太远 | 把舵机往开关方向挪近 1~2mm |
| 杠杆卡住回不来 | 杠杆离开关表面太近，按下后卡住 | 垫高舵机或加厚杠杆端部垫片 |
| 舵机嗡嗡响不动 | 角度超出机械限位，舵机堵转 | 减小 PWM 角度范围，留安全余量 |
| 杠杆按压位置不对 | 杠杆长度或舵机高度不合适 | 微调舵机高度，或换更长/更短的杠杆 |
| 按压时开关发出"咯"声 | 杠杆端部没有软垫片 | 加橡胶垫片，或减薄垫片厚度 |
| 舵机异响发热 | 齿轮卡死 / 堵转 | 检查 PWM 行程是否超过舵机旋转范围 |
| 侧面安装后舵机太长，和开关面板干涉 | 安装空间不足 | 可以把舵机稍微移远一点，杠杆相应加长（杠杆越长，末端行程越大） |

## 九、阶段 7：外壳与最终封装（第 3 周，预计 1~2 天）

### 9.1 外壳设计要点

> ⚠️ **外壳设计依赖于你的开关布局，先完成第 8 节调试，确定舵机安装位置后，再设计外壳。**

1. CubeMX 中开启 **USART2**（PA2_TX, PA3_RX），波特率 115200

3. 如果收到 `OPEN` → 调用 `Servo_SetAngle(DUTY_DOWN)`（开灯）
4. 如果收到 `CLOSE` → 调用 `Servo_SetAngle(DUTY_UP)`（关灯）

## 十一、常见问题排查（Q&A）

| 问题 | 可能原因 | 排查步骤 |
|------|----------|----------|
| **舵机不转** | 接线错误 / PWM 未启动 | 1. 确认信号线接 PA0；2. 检查 `HAL_TIM_PWM_Start` 是否调用；3. 用万用表测 PA0 对 GND 电压（PWM 正常输出时平均电压应在 0.6~1.5V 之间，若为 0V 或 3.3V 说明 PWM 未启动或占空比为 0%/100%）；4. 有条件可用示波器确认 50Hz PWM 波形 |
| **舵机抖动/嗡嗡响** | 供电不足 / 共地问题 | 1. 舵机必须接独立 5V；2. STM32 的 GND 和舵机/JDY-16 的 GND 必须连在一起（共地）；3. 电池电量是否太低（低于 3.0V 时一体板升压效率下降） |
| **舵机角度不对** | CCR 值需微调 | 修改 `DUTY_UP` / `DUTY_DOWN`，每次 ±50 尝试 |
| **手机连不上 JDY-16** | 已被占用 / 未配对 | 1. 重新上电 JDY-16；2. 确认密码 1234；3. 一次只能连一个设备 |
| **串口助手没反应** | 驱动没装 / 波特率不对 | 1. 设备管理器查 COM 口号；2. 确认波特率 115200；3. 重装 CH340 驱动 |
| **编译报 undefined reference** | 缺少外设初始化 | 回到 CubeMX，确认外设已配置，重新 Generate Code |
| **烧录失败** | ST-Link 未连接 / 接线问题 | 1. Keil → Project → Options → Debug → 选 ST-Link Debugger → Settings，看是否显示 IDCODE；2. 检查 SWCLK/SWDIO/GND 接线；3. 确认 STM32 已上电（电池或一体板供电） |
| **接上电池后舵机不动** | 一体板输出异常 | 检查一体板 5V 输出是否正常，一体板无需调电压，出厂即 5V 固定输出。若仍不工作，检查电池是否带保护板、电池电压是否低于 3.0V |
| **充电时系统重启** | 电池电压波动 | 一体板充电时电流较大，确认电池电量不低于 3.0V，或在一体板 5V 输出端加 100µF/25V 电解电容 |
| **续航很短** | 电池容量太小 | 换更大容量 18650 电池（3500mAh 或以上） |

### 13.1 第一梯队：低成本快速实现（1~3 天 / 额外成本 ≈ ¥0~¥10）

#### 1. 电量监测与低电量报警（⭐⭐⭐ 最推荐）

**为什么做**：当前使用 18650 锂电池供电，电压过低会导致舵机转动无力、STM32 重启，用户完全不知道电池快没电了。

**技术方案**：
| 项目 | 说明 |
|------|------|
| 硬件 | 无需额外硬件（STM32F103 自带 ADC） |
| 接线 | 电池正极 → 电阻分压（200k + 100k）→ PA1 (ADC_IN1) |
| 原理 | STM32 ADC 读取 0~3.3V 电压 → 换算成电池电压 |
| 逻辑 | 电压 ≥ 3.7V → LED 常亮绿灯 |
| | 3.2V ~ 3.7V → LED 慢闪绿灯（电量中等） |
| | < 3.2V → LED 快闪红灯 + 蓝牙推送"⚠️ 电量低，请充电" |

**CubeMX 配置**：PA1 配置为 `ADC1_IN1`（模拟输入模式）

**代码提示词（给 AI）**：
```
我在 STM32F103C8T6 上配置了 ADC1 通道 1（PA1）用于读取电池电压，
使用 HAL 库，ADC 分辨率 12 位，参考电压 3.3V。

硬件：电阻分压电路（200k 上拉 + 100k 下拉），分压后 PA1 电压 = 电池电压 × 1/3，即电池电压 = PA1 电压 × 3 = ADC 读数 × 3 × 3.3 / 4096

请帮我写：
1. ADC 初始化代码（单次转换模式）
2. 读取电压并换算的函数
3. 低电量检测逻辑（阈值：3.2V 报警，3.7V 正常）
4. 在 while(1) 中每隔 5 秒检测一次的框架

注意：要加均值滤波（连续采样 10 次取平均），避免电压跳变
```

#### 2. 定时任务功能（⭐⭐⭐ 实用性强）

**为什么做**：很多场景需要定时开关，如"早上 7 点自动开灯"、"睡前 23 点自动关"。

**技术方案**：
| 项目 | 说明 |
|------|------|
| 硬件 | 无需额外硬件（使用 STM32 内部 RTC） |
| 原理 | RTC 实时时钟 + 闹钟中断 |
| 蓝牙协议扩展 | `T,HH:MM,ON` → 设置定时，`T,HH:MM,OFF` → 清除定时 |

**CubeMX 配置**：启用 RTC 功能

> ⚠️ **LSE 晶振提醒**：大多数 STM32F103 蓝色小板**没有板载 32.768kHz 晶振**。在 CubeMX 中如果 LSE 配置失败（显示灰色无法启用），请改用 **LSI（内部低速振荡器，约 40kHz）** 作为 RTC 时钟源。LSI 精度较低（每月偏差约几分钟），对"定时开关灯"场景够用。

**代码提示词（给 AI）**：
```
我在 STM32F103C8T6 上配置了 RTC，
时钟源优先用 LSE（32.768kHz），如果 LSE 不可用则用 LSI（内部约 40kHz）。
使用 HAL 库，格式为 YYYY-MM-DD HH:MM:SS。

请帮我实现定时任务功能：
1. RTC 初始化（从蓝牙指令设置初始时间）
2. 定义一个结构体数组存储最多 3 条定时任务
3. 每分钟检查一次：当前时间是否匹配任何定时任务
4. 匹配时执行对应的开关动作，并打印"定时任务触发"
5. 蓝牙指令解析：收到 'T,07:00,ON' 添加一条早上 7 点开灯的定时

注意：用非阻塞方式实现，不要在 while(1) 里用延时
```

#### 7. FreeRTOS 移植（⭐⭐ 基础设施升级）

**为什么做**：当后续叠加多个升级功能（如定时任务 + 人体感应 + 蓝牙同时运行）时，裸机的 `while(1)` 主循环会变得臃肿难维护。FreeRTOS 可以将各功能拆分为独立任务，互不阻塞，代码结构更清晰。

**技术方案**：
| 项目 | 说明 |
|------|------|
| 硬件 | 无需额外硬件 |
| 成本 | ¥0 |
| 难度 | ⭐⭐（需处理 AC6 编译器兼容性，但有成熟解决方案） |
| 预计时间 | 1~2 天 |

**⚠️ AC6 编译器兼容性处理（必须在移植前完成）**：

CubeMX 生成 FreeRTOS 工程时默认使用 **RVDS 移植文件**（给旧版 AC5 编译器用的），而 Keil MDK 5.37+ 默认使用 **AC6 编译器**（基于 LLVM/Clang），两者不兼容，直接编译会报错。

> **解决方法**：将 FreeRTOS 移植文件从 RVDS 替换为 GCC（AC6 兼容 GCC 格式）

**操作步骤**（一次性修改，永久生效）：

1. 找到 CubeMX 固件库中的 GCC 移植文件：
   ```
   C:\Users\<你的用户名>\STM32Cube\Repository\STM32Cube_FW_F1_Vx.x.x\
   Middlewares\Third_Party\FreeRTOS\Source\portable\GCC\ARM_CM3\
   ```
2. 用该目录下的 `port.c` 和 `portmacro.h` **替换**同级目录中 RVDS 版本的文件：
   ```
   ...\portable\RVDS\ARM_CM3\
   ```
3. 之后每次在 CubeMX 中重新生成工程，都会直接生成 AC6 兼容的文件，无需反复手动替换

> 💡 **原理说明**：AC6 能直接编译 GCC 风格的 C 代码和汇编，所以把 RVDS 的移植层换成 GCC 版本即可。FreeRTOS 内核代码本身是通用的，不需要修改。

**移植步骤概要**：
1. ✅ 先完成上述 RVDS → GCC 文件替换
2. CubeMX 中启用 FreeRTOS：**Middleware → FREERTOS → Interface → CMSIS_V1**
3. 在 Keil 中切换编译器为 **ARM Compiler 6**（见 4.7 节）
4. 创建任务（如 `BluetoothTask`、`ServoTask`），将现有 `while(1)` 中的逻辑拆分到各任务
5. 编译 → 解决可能的头文件路径问题 → 测试

**代码提示词（给 AI）**：
```
我在 STM32F103C8T6 上使用 Keil MDK 5.37+（ARM Compiler 6），
已将 FreeRTOS 移植文件从 RVDS 替换为 GCC 版本。
CubeMX 中已启用 FreeRTOS（CMSIS_V1）。

当前裸机代码功能：蓝牙接收指令 → 解析 → 控制舵机 PWM。
请帮我将代码迁移到 FreeRTOS：
1. 创建蓝牙接收任务（优先级 2）：接收串口数据并解析
2. 创建舵机控制任务（优先级 3）：根据指令控制 PWM
3. 用消息队列传递指令（蓝牙任务 → 舵机任务）
4. 空闲任务中加入低功耗处理

注意：栈大小设置保守一些（各任务 128 words），F103C8T6 只有 20KB RAM
```
